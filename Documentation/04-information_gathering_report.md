# Personal Expense Tracker - Information Gathering Report (Basic)

## 1. Purpose
This report summarizes the basic information gathered to build a simple and effective Personal Expense Tracker for university students and individuals. The goal was to understand daily spending habits and prioritize core features.

## 2. Goals
*   Identify the most common expense categories (Food, Travel, etc.).
*   Understand why manual tracking is difficult for students.
*   Define the simplest way to display financial summaries.

## 3. Methodology Used
| Method | Target Group | Purpose |
| :--- | :---: | :--- |
| **Interviews** | Classmates/Friends | To learn about their daily spending struggles. |
| **Observation** | Self-Checking | Reviewing personal spending for 7 days to identify required data fields. |
| **Reference Study** | Popular Budget Apps | Comparing basic features like balance display and history lists. |

## 4. Key Findings
*   **Quick Entry:** Adding an expense should take less than 10 seconds.
*   **Essential Categories:** Most users only need 5-6 categories (Food, Rent, Bills, Transport, Fun).
*   **Visual Balance:** A clear "Total Balance" display on the home screen is a top priority.
*   **Simple History:** Users want to see their last 5-10 transactions immediately.

## 5. Summary of Required Features
*   **User Login:** To keep individual records separate.
*   **Add Expense:** A simple form with Amount, Category, and Date.
*   **Dashboard:** A visual chart showing spending by category.
*   **Transaction List:** A scrollable list of all previous entries.
