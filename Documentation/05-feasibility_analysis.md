# Personal Expense Tracker - Feasibility Analysis (Basic)

## 1. Technical Feasibility
*   **Tools:** We are using React and FastAPI. Both have lots of tutorials and are easy to learn for university projects.
*   **Database:** PostgreSQL is free and very reliable for storing transaction data.
*   **Conclusion:** It is technically possible to build this with my current skills.

## 2. Economic Feasibility
*   **Software Cost:** $0. Every tool we use is open-source and free to download.
*   **Hardware Cost:** $0. I am using my own laptop for development.
*   **Hosting Cost:** $0. We will use free platforms to host the final project.

## 3. Schedule Feasibility
*   **Week 1:** Setup and Documentation (**Done**).
*   **Week 2-3:** Build the Backend (API and Database).
*   **Week 4-5:** Build the Frontend (UI and Charts).
*   **Week 6:** Test and fix bugs.
*   **Conclusion:** There is plenty of time to finish before the semester ends.

## 4. Risks and Solutions
*   **Risk:** Running out of time. *Solution:* Focus on the core "Add Expense" feature first.
*   **Risk:** Bugs in calculations. *Solution:* Test with simple numbers to ensure totals are correct.
*   **Risk:** Data privacy. *Solution:* Use a simple login system for every user.
