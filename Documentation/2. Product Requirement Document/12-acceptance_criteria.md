# 12 - Acceptance Criteria 

## Feature 1: User Login
- A user must be able to sign up with email and password.
- A user must be redirected to the dashboard after a successful login.

## Feature 2: Add Transaction
- The amount field must only accept numbers.
- The user must select a category before saving.
- The transaction must show up in the history list immediately.

## Feature 3: Dashboard View
- The 'Total Balance' must update automatically after each change.
- The Pie Chart must show at least 3 categories (Food, Transport, Others).

## Feature 4: History List
- Transactions must be sorted by date (newest first).
- Each item must show the Amount, Category, and Date.
