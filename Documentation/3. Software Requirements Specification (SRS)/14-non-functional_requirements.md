# 14 - Non-Functional Requirements

Non-functional requirements define how the system should perform (Speed, Security, Reliability).

| ID | Category | Requirement |
|---|---|---|
| **NFR-01** | **Performance** | Logging an expense should take less than 3 seconds. |
| **NFR-02** | **Security** | User passwords must be encrypted before being stored in the database. |
| **NFR-03** | **Scalability** | The system should support at least 100 concurrent users without slowing down. |
| **NFR-04** | **Availability** | The web app should be available 99.9% of the time. |
| **NFR-05** | **Usability** | The interface must be mobile-responsive and easy to navigate on a small screen. |
| **NFR-06** | **Privacy** | A user should never be able to see another user's financial data. |
