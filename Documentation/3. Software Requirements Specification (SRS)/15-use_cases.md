# 15 - Use Cases

## Use Case 1: Add a New Expense
- **Actor:** User
- **Pre-condition:** User is logged in.
- **Main Flow:**
  1. User clicks the "Add Transation" button.
  2. User enters the Amount (e.g., 50).
  3. User selects a Category (e.g., Food).
  4. User clicks "Save".
  5. System updates the Balance and adds the item to the History.
- **Post-condition:** Transaction is saved successfully.

## Use Case 2: View Spending Analysis
- **Actor:** User
- **Main Flow:**
  1. User navigates to the Dashboard.
  2. System fetches all transactions for the current month.
  3. System displays a Pie Chart showing the spending distribution.
- **Goal:** User understands their spending habits visually.
