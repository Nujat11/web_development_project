# 13 - Functional Requirements

Functional requirements define what the system must do. These are derived from the user stories.

| ID | Feature | Description |
|---|---|---|
| **FR-01** | User Registration | System must allow users to create an account with email and password. |
| **FR-02** | User login | System must verify user credentials and allow access. |
| **FR-03** | Add Transaction | System must allow users to log an amount, category, and date. |
| **FR-04** | Categorization | System must provide a list of categories (Food, Rent, Travel, etc.). |
| **FR-05** | Balance Calculation | System must automatically calculate the total balance in real-time. |
| **FR-06** | History List | System must display a chronological list of all saved transactions. |
| **FR-07** | Pie Chart | System must generate a pie chart based on spending categories. |
| **FR-08** | Data Editing | System must allow users to edit or delete their own transactions. |
| **FR-09** | Logout | System must securely end the user session. |
